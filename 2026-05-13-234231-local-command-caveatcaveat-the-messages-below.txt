  1. 推荐：给卡牌配置增加特效 Addressables Key

    - 优点：不用改配置
    - 缺点：会在表现层塞对象池逻辑，不如 ResManager 统一

    - 优点：最少改动
    - 缺点：连续特效多时性能差，和你 ResManager

  因为现在 CardDataSO.CardEffectPrefab 已经是 GameObject
  预制体引用，如果强行改成 Addressables key，还要改配置表/资源配置。在 
  ResManager 里补一套“按 prefab

  ResManager，比把池写进 PlayEventSequence 干净。

  推荐设计：

  - ResManager 保留现有 string keyName 池，继续给 Addressables 用
  - 新增一个 Dictionary<GameObject, Queue<GameObject>>，专门按 prefab 引用池化
  - 新增方法：
    - InstantiateFromPool(GameObject prefab, Transform parent = null)
    - ReleaseToPool(GameObject prefab, GameObject obj, int maxPoolSize = 20)
    - 可选：ClearPrefabPool(GameObject prefab) / ClearAllPrefabPools()
  - PlayEventSequence.tryPlayCardVfx() 改成：
    - 从 cardData.CardEffectPrefab 池里取
    - 设置位置：target.transform.position + Vector3.up * 1.25f
    - 3 秒后回收到 prefab 池

  GameObject vfx = ResManager.InstantiateFromPool(cardData.CardEffectPrefab);
  vfx.transform.SetPositionAndRotation(spawnPos, Quaternion.identity);

  DOVirtual.DelayedCall(3f, () =>
  {
      ResManager.ReleaseToPool(cardData.CardEffectPrefab, vfx);
  });

  这个方案的优点：

  - 不用改 CardDataSO
  - 不用重新配置 Addressables key
