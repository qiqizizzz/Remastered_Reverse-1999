# 🌟 Remastered Reverse: 1999 (重返未来1999 复刻版)

## 📖 项目简介
本项目是一个基于《重返未来：1999》核心玩法的复刻工程。项目涵盖了从底层架构、数据库持久化、网络通信到核心战斗系统的前后端完整闭环。整体不仅采用了 **MVC 架构**，更在核心战斗流中深度应用了多种**面向对象设计模式（如命令模式、状态模式）**，是一个具备极高技术参考价值的单人全栈游戏开发实践。

---

## 🛠️ 技术栈与核心架构
* **整体架构**：MVC (Model-View-Controller) 设计模式，确保数据、逻辑与表现的解耦。
* **战斗架构**：
  * **命令模式 (Command Pattern)**：卡牌的所有操作被严格封装为命令对象，完美支持动作队列处理、复杂结算顺序及前后端状态同步。
  * **状态模式 (State Pattern)**：PVE 玩法中，所有实体角色（敌方与英雄）的逻辑皆使用状态机（FSM）进行封装，使得状态切换（受击、施法、待机等）清晰且易于扩展。
* **前后端与存储**：基于本地服务端与客户端（C/S 架构）实现，并接入 **MySQL** 实现真正的用户数据持久化存储。
* **数据通信**：使用 Protocol Buffers (Protobuf) 进行高效的序列化与网络数据交互。
* **资源管理**：基于 Addressables (AA包) 与对象池（Object Pool）的高效 `ResManager`。

---

## ✨ 核心功能模块

### 1. 基础网络与账号系统
* **服务端与数据库**：实现了完整的 C/S 数据交互，借助 MySQL 落库，确保玩家数据的安全与持久化。
* **登录与注册功能**：基于 Protobuf 协议处理账号的鉴权与数据传输。

### 2. 核心战斗玩法
* **卡牌战斗系统**：深度复刻《重返未来：1999》的回合制卡牌核心机制。得益于命令模式与状态模式的结合，战斗结算严谨，易于维护。
* **多模式支持**：同时搭建了 **PVE**（玩家对战环境）与 **PVP**（玩家对战玩家）双端战斗逻辑。

### 3. UI 与交互系统
* **角色界面（无限列表）**：采用无限滚动列表（Infinite Scroll/Recycling List）技术，完美解决海量角色数据的渲染性能瓶颈。
* **设置模块**：包含音量调节等完善的全局游戏选项控制。
* **聊天系统**：实现了玩家间的基础文本交互功能。

### 4. 通用系统与优化组件
* **简易红点系统**：巧妙运用**前缀树（Trie）**数据结构，实现了层级分明、状态更新高效的红点提示功能。
* **资源管理器 (ResManager)**：结合 AA包（Addressables）进行异步加载，并封装底层对象池（Object Pool），大幅降低内存开销与 GC 频率。
